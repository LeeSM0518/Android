

# 12. 메시징과 소셜 네트워크 서비스

# 12-1. 메시징 서비스 이해하기

메시지를 서로 주고받을 수 있는 서비스를 흔히 **메시징 서비스(Messaging Service)** 라 부르는데 메시징 서비스는 다음과 같이 크게 네 가지로 나눠 볼 수 있다.

* **메시징 서비스**

  1. **단말 간 메시지 송수신 (SMS/ MMS)**

     : 이동통신사의 서버를 사용한다.

  2. **서버를 통한 단말 간 메시지 송수신 (XMPP, SIP, KakaoTalk)**

  3. **SNS를 통한 단말 간 메시지 공유 (Twitter, Facebook)**

  4. **푸시 서비스를 통한 메시지 전송 (FCM)**



# 12-2. 푸시 서비스 사용하기

**푸시 메시지란** 단말의 위쪽 부분에 보이는 상태바 부분에 업데이트 요청과 같은 메시지.

* **푸시 알림 메시지를 보내는 기술적인 방법**

  | 속성                                    | 설명                                                         |
  | --------------------------------------- | ------------------------------------------------------------ |
  | 단순 SMS를 이용한 알림                  | 간단하지만 비용이 발생한다.                                  |
  | 앱에서 연결을 만들어 놓은 상태에서 알림 | 앱에서 서버와의 연결을 만들어 놓고 폴링하는 과정이 필요하며, 백그라운드 서비스를 이용해야 한다. |
  | 구글의 푸시 서비스(FCM)를 사용하여 알림 | 앱에서 직접 연결할 필요가 없으며 단말의 내부 연결을 공유하여 데이터를 수신하는 방식이다. |



## 12-2. 푸시 서비스 이용(예제)

### FCM 설정 페이지에서 프로젝트 만들기

[Firebase 개발자 콘솔(Console) 페이지](https://console.firebase.google.com)

위의 페이지에 접속하여 [새 프로젝트 만들기] 버튼을 누르면 [프로젝트 만들기] 대화상자가 나타난다. 그러면 '프로젝트 이름' 입력 상자에 **프로젝트 이름을 입력하고 애널리틱스 위치를 대한민국으로 변경한다.**
